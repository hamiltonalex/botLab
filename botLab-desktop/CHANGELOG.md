# Changelog

Все заметные изменения BotLab фиксируются в этом файле.
Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/); версии — [SemVer](https://semver.org/lang/ru/).

Раздел версии соответствует телу релиза на GitHub. Строка «Влияние» в шапке релиза оценивает
последствия обновления по областям: торговая логика · P&L · UI · настройки/данные.

Скоупы изменений: **[App]** — оболочка (апдейтер, справка, журнал-инфраструктура, релизный конвейер);
**[Funding-Arb]** — бот арбитража ставок финансирования; **[BTC-Options]** — бот опционных структур.

## [Unreleased]

**Влияние: торговая логика · P&L · UI** — пресеты дедбэнда и выбор исполнения теперь реально
управляют хедж-движком (ширина коридора и цена/комиссия бумажного исполнения меняются); структура,
дожившая до экспирации, рассчитывается автоматически по внутренней стоимости вместо замирания.
Funding-arb не затронут.

### Добавлено

- [BTC-Options] Расчёт на экспирации: дожившая до экспирации структура автоматически
  рассчитывается по внутренней стоимости на цене индекса (приближение расчётной цены Deribit,
  зафиксировано в примечании строки журнала), перпетуал выравнивается, в журнале — событие
  «расчёт на экспирации». Ранее позиция замирала: ноги исчезали из данных, гейт греков падал,
  а переоценка показывала ложный ноль.
- [BTC-Options] Метрики прогона переживают следующую сделку: при закрытии или расчёте структуры
  сводка завершённого прогона замораживается; экспорт JSON и XLSX содержит метрики текущего и
  последнего завершённого прогонов (CSV остается чистой таблицей событий).
- [BTC-Options] Строка «гейт греков не пройден» в тикете называет инструменты, по которым не
  хватает данных; полный список и детали ошибок — в подсказке при наведении.

### Исправлено

- [BTC-Options] Пресет дедбэнда меняет фактическую ширину коридора (агрессивный ±0.0005 BTC,
  нормальный ±0.001, консервативный ±0.002), а не только подпись: ранее движок продолжал работать
  шириной по умолчанию при любом выборе. Несогласованные пары пресет/ширина в сохранённых
  настройках разово выправляются при запуске; параметры открытой структуры, как и прежде,
  заморожены до следующего открытия.
- [BTC-Options] Выбор «Исполнение» доезжает до хедж-движка и честно моделируется: рыночное —
  пересечение спреда с комиссией 0.05%, лимитное — исполнение по середине спреда с комиссией 0%
  (компромисс за немоделируемый риск неисполнения); проскальзывание сохранено в обеих ветках как
  ненулевой пол издержек. Ранее выбор не покидал конструктор, а все исполнения шли по рыночной
  модели.
- [BTC-Options] Сверка журнала (ledgerReconciles) не требует строк фандинга (начисление —
  сознательно только в аккумуляторе) и сводит реализованный результат против суммы обоих
  аккумуляторов — перпа и опционов; ранее проверка гарантированно расходилась на любом прогоне
  с фандингом или после первого закрытия структуры.
- [BTC-Options] Временной триггер в карточке «Хедж-движок · решение» больше не называется
  «интервалом реприса»: подпись говорит «интервал хеджа» и показывает его значение (60 с
  по умолчанию). Это отдельный параметр движка, не связанный с интервалом опроса источника;
  справка панели уточнена.
- [BTC-Options] Интервал реприса по умолчанию — 15 секунд, как в селекторе (5/15/30). Ранее
  скрытый дефолт равнялся 3 секундам: селектор не мог его ни показать (ни одна кнопка не
  подсвечивалась), ни вернуть после первого изменения. Профили со старым дефолтом разово
  выправляются при запуске; выбранные вручную значения не затрагиваются.
- [BTC-Options] Сбой вспомогательного инструмента (дежурный набор для сигнала IV) больше не
  переводит статус источника из **LIVE** в **ПРЕДУПР.**: здоровье источника оценивается по
  перпетуалу и ногам открытой структуры, как и задумано; ошибки band-инструментов остаются
  видимыми в подсказке статуса. Гейт греков и логика хеджа и ранее оценивались корректно —
  деградировал только бейдж.
- [BTC-Options] Открытие структуры защищено от гонки котировок: страйки теперь выбираются по той
  же цене, по которой опрашивались ноги, а нога без котировки стала блокирующей причиной
  «нет котировки (mark)» в тикете. Ранее при движении цены между подбором ног и открытием
  структура могла открыться с непрокотированной ногой, и её премия молча выпадала из
  нетто-дебета и оценки максимального убытка.
- [BTC-Options] Закрытие структуры при снимке без цены перпетуала откладывается с ошибкой
  «нет цены перпетуала в снимке», а не исполняется наполовину. Ранее опционная часть
  фиксировалась, структура очищалась, а невыровненный хедж превращался в неуправляемую
  позицию: кнопка закрытия прячется вместе со структурой, фандинг продолжал начисляться.

## [0.3.0] — 2026-07-12

Крупный выпуск: BotLab становится мультиботовой оболочкой, появляется второй бот «BTC-опционы»
(бумажный режим) и светлая тема оформления.

**Влияние: торговая логика · UI · настройки/данные** — добавлен движок опционных структур с
дельта-хеджем (только бумажный тест, реальные ордера не выставляются); интерфейс перестроен в
оболочку с вкладками, добавлена светлая тема. Настройки расширены (тема интерфейса, пофайловые
настройки ботов). Логика и P&L funding-arb не изменены, существующие позиции и журнал не затронуты.

### Добавлено

- [App] Оболочка-переключатель модулей: верхние вкладки «Обзор · Funding-arb · BTC-опционы» и
  стартовый экран «Обзор» со сводкой по ботам (готовые модули, открытые позиции, суммарный P&L).
  Funding-arb вынесен в отдельный вид без изменения его логики; навигация — по паттерну
  WAI-ARIA tabs с полной поддержкой клавиатуры.
- [App] Светлая тема оформления: сегмент-переключатель в шапке — луна/солнце, активная тема
  подсвечена; выбор сохраняется в настройках и применяется до первой отрисовки окна — без вспышки
  при старте. Палитра «бумажный
  терминал» сохраняет идентичность тёмной: те же оттенки зон и семантики прибыль/убыток/нейтрально,
  контраст ключевых пар текст/фон не ниже WCAG AA; графики перерисовываются в цветах темы при
  переключении. Тёмная тема осталась прежней и по умолчанию.
- [BTC-Options] Новый бот «BTC-опционы»: крылатый стрэддл на линейных USDC-опционах BTC
  (длинные ATM колл и пут, короткие OTM-крылья, один экспиратор не позднее 3 дней) с дельта-хеджем
  инверсным перпетуалом BTC-PERPETUAL. Данные — публичный REST Deribit, только чтение, без ключей
  и без ордеров (paper); бумажный депозит по умолчанию 100 USD. Движок изолирован от funding-arb:
  отдельный IPC-мост и отдельные файлы состояния.
- [BTC-Options] Хедж-движок: триггеры дельта/цена/время, фильтр издержек (сделка только при
  ожидаемой выгоде не ниже λ·издержек), пауза в окне расчёта 08:00 UTC и в последние 30 минут перед
  экспирацией. Панель решения (ХЕДЖ / ПРОПУСК / ПАУЗА), разнесённый P&L (MTM опционов, реализованный
  хедж, комиссии, фандинг перпа, нетто), таблица четырёх ног с греками, диаграммы payoff и
  дельта-коридора, журнал операций с экспортом CSV/XLSX/JSON.
- [BTC-Options] Сравнение «хедж против без-хеджа»: параллельный теневой расчёт без перпа показывает,
  окупает ли хедж собственные издержки.
- [BTC-Options] Метрики прогона: Sharpe по цикловым доходностям, доля прибыльных циклов,
  максимальная просадка, число и средний размер хеджей; графики P&L-траектории с атрибуцией и
  справочной цены BTC с маркерами исполнения хеджей.
- [BTC-Options] Реальная маржа Deribit (стандартная маржа коротких ног, без неттинга): начальная и
  поддерживающая маржа, текущая и худшая утилизация, честный флаг превышения депозита и алерт
  поддерживающей маржи с порогом 80 %.
- [BTC-Options] Стресс-сценарии: сдвиги цены ±5 % и ±10 %, скачок IV ±25 vol, фандинг ×3 за горизонт;
  каждая строка помечена режимом оценки (мгновенный / на экспирацию / за горизонт), сдвиги цены
  дублируются метками на диаграмме payoff.
- [BTC-Options] Авто-конструирование: кнопка «Старт (авто)» сама выбирает ближайшую живую экспирацию
  (не позднее 3 дней, минуя предэкспирационное окно) и страйки по офсету; подтверждение — тем же
  тикетом. Структурированные пре-трейд проверки: блокирующие причины (минимальный лот, кратность
  шагу, блэкаут расчёта) запрещают открытие; предупреждение «начальная маржа больше депозита»
  открытие не блокирует — решение остаётся за пользователем.
- [BTC-Options] Сигнал входа по волатильности: источник опрашивает ATM-полосу (стрэддл и крылья
  ±5/10/15 %), IV-ранг считается по скользящему окну 24 часа, индекс DVOL кэшируется с 48-часовой
  историей; карточка «IV-режим · сигнал входа» отвечает, благоприятен ли вход сейчас.
- [BTC-Options] Свип параметров: честный реплей накопленной истории снапшотов через чистый движок по
  сетке «крылья × дедбэнд × ценовой триггер × λ» (108 комбинаций), ранжирование по Sharpe, отметка
  комбинаций, не помещающихся в депозит; двухшаговое применение лучших параметров к конструктору
  (открытая структура и её замороженные параметры не затрагиваются).
- [BTC-Options] Телеметрия REST-канала: замер круговой задержки и серверного времени биржи; штамп
  подключения показывает канал и RTT. WebSocket-транспорт оценён и отложен: REST не показывает
  давления на лимиты, шов источника оставлен готовым к замене.

### Изменено

- [App] Заголовок окна «Funding-Arb Paper Simulator» → «BotLab».
- [App] «Обзор» суммирует оба бота (готовые модули, открытые позиции и структуры, суммарный P&L).
  Кластер интервала опроса и LIVE стал специфичным для funding-arb; у BTC-опционов собственный
  кластер подключения к Deribit.

### Исправлено

- [BTC-Options] Тринадцать исправлений по итогам pre-merge ревью и сквозного UI-прогона (2026-07-11).
  Критические: изменение настроек при работающем источнике останавливало опрос Deribit до
  перезапуска приложения; гейт греков пропускал хедж по заниженной дельте при полностью выпавшей из
  снапшота ноге; повторное открытие структуры молча затирало текущую вместе с её нереализованным
  результатом. Существенные: неверный знак фандинга в модели издержек хеджа для короткой целевой
  позиции; перекрывающиеся тики REST-источника могли записывать устаревшие снапшоты поверх свежих;
  кнопка подтверждения тикета оставалась активной после блокирующего отказа движка; превью
  конструктора не пересчитывалось после закрытия структуры; свип исключал все комбинации, если
  источник был запущен до первого обращения к цепочке инструментов; применённые настройки не
  доходили до движка до перезапуска. Прочие: Escape закрывает тикет опционов из любого фокуса;
  причины пре-трейд проверки экранируются перед выводом; точки безубытка не рисуются за пределами
  крыльев, где кривая уже плоская; поллинг ATM-полосы больше не останавливается после закрытия
  структуры.

## [0.2.0] — 2026-07-05

Первый публичный выпуск: приложение переименовано в **BotLab** и получило обновления «по воздуху».

**Влияние: UI · настройки/данные** — переименование переносит каталог `userData` (позиции, журнал и
настройки мигрируют автоматически). Торговая логика и P&L не изменены.

### Добавлено

- [App] Обновления «по воздуху» из GitHub Releases: пилюля версии в шапке с восемью состояниями,
  скачивание только по клику, тихая установка с перезапуском, проверка при старте и каждые 6 часов.
- [App] После обновления страница релиза открывается в браузере один раз; «Что нового» и
  «Показать лог» доступны из поповера пилюли.
- [App] Целостность: SHA-512 каждого скачивания сверяется с манифестом релиза; при несовпадении
  установка не начинается.
- [App] Справка стала модульной (ядро/бот) с новой записью «Обновления приложения»; покрытие справки
  и состояния пилюли обновлений проверяются оракулом.
- [App] Релизный конвейер GitHub Actions: тег `vX.Y.Z` → тесты → сборка → подпись → нотарификация →
  черновик релиза (публикация — вручную после дымовой проверки).

### Изменено

- [App] Приложение переименовано «Funding-Arb Paper Simulator» → **BotLab**; `appId` →
  `com.hamiltonalex.botlab`. После этого релиза имя, `appId` и Team ID заморожены (идентичность
  апдейтера и подписи).

### Исправлено

- [App] Заметки релиза в поповере пилюли теперь отображаются как читаемый текст: GitHub отдаёт их в
  HTML, из-за чего раньше в поповере были видны теги.
- [App] Релизный CI: тесты запускаются через `node --test` (прежний вариант с путём `test/` падал на
  Windows-раннере, загружая каталог как модуль).

### Известные проблемы

- [App] Этот выпуск — только Windows. Сборка для macOS (с подписью Developer ID и нотарификацией)
  выйдет следующим релизом после настройки сертификата Apple.
- [App] Windows-сборка пока не подписана: SmartScreen может предупредить при первичной установке
  (тихие обновления обычно не затрагиваются). Подпись — в следующих релизах.
- [App] Тестерам версии 0.1.0: переход на BotLab 0.2.0 — ручной (установить новое приложение, затем
  удалить старое). Позиции, журнал и настройки подхватятся автоматически при первом запуске BotLab;
  старый каталог данных остаётся как страховка отката, пока старое приложение не удалено.
