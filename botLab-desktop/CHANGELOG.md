# Changelog

Все заметные изменения BotLab фиксируются в этом файле.
Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/); версии — [SemVer](https://semver.org/lang/ru/).

Раздел версии = тело соответствующего релиза на GitHub (§8.2). Каждый релиз в шапке несёт строку
**Влияние** — осознанная оценка последствий обновления:

- ⚠️ торговая логика · 💰 P&L · 🖥 UI · ⚙️ настройки/данные · «Влияние: нет»

Скоупы изменений: **[App]** — оболочка (апдейтер, Help, журнал-инфраструктура, релиз);
**[Funding-Arb]** — бот арбитража ставок финансирования. Новые боты добавляют свой скоуп.

## [Unreleased]

### Добавлено
- [App] Оболочка-переключатель модулей: верхние вкладки «Обзор · Funding-arb · BTC-опционы» и
  стартовый экран «Обзор» с карточками ботов и сводкой счёта (готовых модулей, открытых позиций,
  суммарный P&L · нетто). Funding-arb вынесен в отдельный вид без изменения его логики. Навигация —
  по паттерну WAI-ARIA tabs (клавиатура ←/→/Home/End, управление фокусом, учёт
  `prefers-reduced-motion`). Реестр ботов data-driven.
- [BTC-Options] Каркас второго бота «BTC-опционы» (стратегия Дмитрия Маринкина №1: крылатый стрэддл на
  BTC-опционах + дельта-хедж перпетуалом; живые публичные данные Deribit, read-only, без ключей и
  ордеров — paper). **Фаза 0 — изоляция логики:** отдельный чистый движок (`src/engine/btcopt/`),
  параллельный IPC-мост `window.s1` (мост `fa` не тронут), своё состояние в отдельных файлах
  (`btc-options.json`), не затрагивающее файлы funding-arb. UI собирается по фазам; заглушка
  «Basis-carry · СКОРО» заменена реальным модулем.
- [BTC-Options] **Фаза 1 — рабочее ядро (MVP): живые данные Deribit + хедж-движок + полный вид.**
  Структура — крылатый стрэддл на линейных USDC-опционах BTC (`BTC_USDC-*`, контракт 1 BTC, мин. лот 0.01):
  длинные ATM колл+пут, короткие OTM колл (Kc≈+10%) и пут (Kp≈−10%), один экспиратор ≤3д; дельта-хедж
  инверсным перпетуалом `BTC-PERPETUAL` ($10). Данные — публичный REST Deribit через `fetch` (без ключей и
  ордеров — paper); источник владеет своим таймером и работает только между «пуск» и «стоп».
  **Хедж-движок:** пересчёт греков на тике → стек триггеров (дельта · цена · время) → фильтр издержек (хедж
  только если ожидаемая выгода ≥ λ·издержки, λ≈1.25) → блэкаут расчёта (пауза в окне 08:00 UTC и последние
  30 мин до экспирации). **P&L разнесён** (MTM опционов · реализованный хедж · комиссии · фандинг перпа ·
  нетто) со сверкой; инверсная переоценка перпа и его фандинг локализованы. **UI `#view-btc-options`:** две
  зоны (Хедж-движок / Конструктор), панель решения движка (триггеры → фильтр → ХЕДЖ/ПРОПУСК/ПАУЗА + ордер,
  `aria-live`), кокпит P&L-атрибуции, таблица 4 ног с греками, диаграммы payoff и дельта-коридора, форма
  открытия структуры с превью-гипотезой, журнал хеджей (CSV/XLSX/JSON), свой кластер подключения к Deribit.
  Бумажный депозит по умолчанию $100. Движок чист и детерминирован (время — через `nowMs`); тесты на
  записанных снапшотах воспроизводят числа спецификации (нетто-дельта −0.0019; фильтр 0.63 &gt; 0.126·1.25;
  блэкаут; кусочный payoff; сверка P&L).
- [BTC-Options] **Фаза 2a — хедж vs без-хеджа.** Теневой прогон без хеджа (перп обнулён) даёт истинное
  сравнение «только опционы, после издержек»: нетто тени равен MTM опционов, а вклад хеджа
  `= реализ.перп + фандинг − комиссии` показывает, окупил ли хедж свои издержки (вердикт «улучшил/ухудшил
  результат после издержек»). Новая панель «Хедж vs без-хеджа»; справка `opt-hedgevs`.
- [BTC-Options] **Фаза 2b — метрики прогона и графики.** Инкрементные O(1)-аккумуляторы в состоянии
  (чистый модуль `metrics.js`): Sharpe по цикловым доходностям (цикл = тик реприса), доля прибыльных,
  макс. просадка, число и средний размер хеджей, наибольшее Δ-отклонение до хеджа; метрики
  сбрасываются при открытии структуры. Два канваса на общем кольце: «P&L-траектория · атрибуция»
  (нетто + опционы/перп/комиссии/фандинг) и «Справочная цена BTC» с маркерами исполнения хеджа.
  Справки `opt-metrics`, `opt-price`.
- [BTC-Options] **Фаза 2c — реальная маржа и алерты риска.** Начальная (IM) и поддерживающая (MM) маржа
  коротких ног по публичным формулам Deribit (линейные USDC BTC: 0.15/0.10/0.075, стандартная маржа без
  неттинга; длинные ноги маржу не требуют) заменяют дебет-прокси Фазы 1 — чистый модуль `margin.js`.
  Утилизация (текущая/худшая), честный флаг превышения депозита (IM&gt;equity) и алерт поддерж. маржи
  (порог 80%). На малом размере IM крылатого стрэддла (~$122) превышает депозит $100 — показано честно,
  но открытие не блокируется (paper). Панель «Маржа и риск»; справка `opt-margin`.
- [BTC-Options] **Фаза 2d — стресс-сценарии.** Чистый модуль `stress.js`: сдвиги цены (±5% / хвост ±10%)
  по гибриду «мгновенная гамма ↔ предел payoff на экспирацию» (оценка никогда не превышает выплату
  крыльев), скачки IV `ΔV≈net_vega·ΔIV` (±25 vol) и стресс фандинга ×3 за горизонт — каждый ряд помечен
  режимом (мгновенный / на экспирацию / за горизонт). Несётся в `cycle.stress`; таблица «Стресс-сценарии»
  + наложение синих меток сдвигов на диаграмму payoff. Справка `opt-stress`.
- [BTC-Options] **Фаза 3a — авто-конструирование + структурированные пре-трейд проверки.** Кнопка
  «▶ Старт (авто)»: движок сам выбирает ближайшую живую экспирацию ≤3д (`pickExpiry`, минуя окно
  &lt;30 мин до экспирации — авто никогда не открывается в распад дельты) и ATM/крылья по офсету;
  подтверждение — тем же тикетом, конструктор синхронизируется с выбором (пилюля «эксп. авто → …»).
  Пре-трейд гейт при открытии (`preTradeCheck`): минимальный лот / кратность шагу / блэкаут расчёта —
  «✕ блок» (открытие запрещено, причина в тикете), реальная начальная маржа Deribit против депозита —
  «⚠ предупр.» («IM $122 &gt; депозит $100»), открытие не блокируется (решение пользователя, паритет с 2c).
  Причины структурированы (`rejections[{code,severity,detail}]`) и показываются в тикете ДО подтверждения
  (превью) и ПОСЛЕ отказа движка. Справка `opt-auto`; S1_SMOKE переведён на полный авто-цикл.
- [BTC-Options] **Фаза 3b — IV-режим входа + систематический свип параметров.** Источник теперь опрашивает
  ATM-полосу (ATM-стрэддл + крылья ±5/10/15% ∪ ноги открытой структуры), поэтому сигнал входа жив и во
  флэте, а история снапшотов несёт котировки всех крыльев (гейт греков остался строго по ногам открытой
  структуры). Ограниченные кольца истории в main-процессе: IV-история (шаг 30с, ~24ч, своя
  `btc-options-history.json`, переживает рестарт) и история снапшотов (600 тиков, сессионная) — состояние
  движка по-прежнему O(1) на тик. Чистый `regime.js`: IV-ранг `(IV−min)/(max−min)` по скользящему окну,
  вход благоприятен при ранге ≤ `ivEntryMaxRank` (0.35) — вместо жёсткого «IV ниже 25»; DVOL
  (`public/get_volatility_index_data`) кэшируется и подтягивает 48ч истории, DVOL-ранг осмыслен с первых
  минут; `cycle.iv_regime` + карточка «IV-режим · сигнал входа». Чистый `sweep.js` (`s1:runSweep`):
  честный реплей накопленного ряда через свежий движок на комбинацию «крылья × дедбэнд × триггер × λ»
  (108), ранжирование по Sharpe (метрики — те же `metrics.js`), комбинации без котировок крыла в ряде
  исключаются, IM>депозита помечается; таблица «Свип параметров» + двухшаговое «применить лучшие
  параметры» (пишет только конструктор Зоны Ⅰ). Справки `opt-iv`, `opt-sweep`; DVOL-проба в
  `smoke-deribit`; S1_SMOKE гоняет свип на живой истории.
- [BTC-Options] **Фаза 3c — телеметрия REST; WebSocket-транспорт оценён и ОТЛОЖЕН.** Решение: WS не
  строится — REST не показывает давления (5 GET/тик при каденции 3с ≈ 1.7 req/с, без 429 за Фазы 1–3;
  цепочка и так на 5-мин кэше), а Node 20 (Electron 33 main и системный) не имеет глобального WebSocket —
  WS потребовал бы зависимость `ws`. Вместо этого закрыт пробел в доказательствах: `rpc` замеряет
  round-trip и серверное время биржи (`usDiff`) → `status().rttMs/usDiffMs` → штамп подключения показывает
  «канал REST» и RTT-подсказку; поля добавлены в S1_SMOKE. Шов `MarketSource` документирован готовым к
  `createWsSource`, если живое использование покажет лаг.

### Изменено
- [App] Заголовок окна «Funding-Arb Paper Simulator» → «BotLab».
- [App] «Обзор» суммирует оба бота (готовые модули · открытые позиции/структуры · суммарный P&L). Кластер
  `#botTools` (интервал опроса, LIVE) стал специфичным для funding-arb — у BTC-опционов свой in-view кластер
  подключения, чтобы контролы `fa.*` не появлялись на чужом виде.

## [0.2.0] — 2026-07-05

Первый публичный выпуск: приложение переименовано в **BotLab** и получило обновления «по воздуху».

**Влияние: 🖥 UI · ⚙️ настройки/данные** — переименование переносит каталог `userData` (позиции,
журнал и настройки мигрируют автоматически). Торговая математика не изменена (⚠️/💰 — нет).

### Добавлено
- [App] Обновления «по воздуху» из GitHub Releases: пилюля версии в шапке с 8 состояниями,
  скачивание только по клику, тихая установка с перезапуском, проверка при старте и каждые 6 часов.
- [App] После обновления страница релиза открывается в браузере один раз; «Что нового» и
  «Показать лог» доступны из поповера пилюли.
- [App] Целостность: SHA-512 каждого скачивания сверяется с манифестом релиза; при несовпадении
  установка не начинается.
- [App] Help стал модульным (ядро/бот) с новой записью «Обновления приложения»; регрессия
  покрытия (`helpCoverage`) и состояний пилюли (`updaterStates`) — в оракуле.
- [App] Релизный конвейер GitHub Actions: тег `vX.Y.Z` → тесты → сборка → подпись → нотарификация →
  черновик релиза (публикация — вручную после дымовой проверки).

### Изменено
- [App] Приложение переименовано «Funding-Arb Paper Simulator» → **BotLab**; `appId` →
  `com.hamiltonalex.botlab`. После этого релиза имя, `appId` и Team ID заморожены (идентичность
  апдейтера и подписи).

### Исправлено
- [App] Заметки релиза в поповере пилюли теперь отображаются как читаемый текст: GitHub отдаёт их в
  HTML, из-за чего раньше в поповере были видны теги (`<p>…</p>`).
- [App] Релизный CI: тесты запускаются через `node --test` (прежний `node --test test/` падал на
  Windows-раннере, загружая `test/` как модуль).

### Известные проблемы
- [App] Этот выпуск — **только Windows**. macOS-сборка (с подписью Developer ID + нотарификацией)
  выйдет следующим релизом, как только будет настроен сертификат Apple.
- [App] Windows-сборка пока не подписана: SmartScreen может предупредить при **первичной** установке
  (тихие обновления обычно не триггерят). Подпись — в следующих релизах.
- [App] Тестерам версии 0.1.0: переход на BotLab 0.2.0 — **ручной** (установить новое приложение,
  затем удалить старое). Позиции, журнал и настройки подхватятся автоматически при первом запуске
  BotLab; старый каталог данных остаётся как страховка отката, пока вы не удалите старое приложение.
